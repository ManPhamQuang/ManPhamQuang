Project Idea

Raspberry Pi Web Server

Overview

Traditionally, hosting websites would require dedicated web servers, or using a VPS (Virtual Private Server) to save hardware expense. While the majority of small businesses use VPS, it is still quite expensive for most students or freelancers like me to afford.

Thankfully, The Raspberry Pi Foundation recently released their newest entry to the micro-computers line of products: Raspberry Pi 4 Model B 2019. Being a microcomputer, it is very budget friendly for students. This would be my project idea, to use and configure a Raspberry Pi to host multiple websites, saving the cost of renting vps for each and every website.

Motivation

Generally, micro-computers are not designed to be dedicated servers, these are low powered, slow, not properly cooled and therefore, not suitable for running multiple services 24/7. Everything changed with the release of the Raspberry Pi 4 Model B 2019, with 4GB of Memory and the new quad core cpu, one can theoretically host up to 6 different Node js websites with memory to spare. Additionally, the Pi 4 still run with low power making it much more efficient and practical than a dedicated server machine which hardware may or may not be utilized to the maximum.

With the Pi 4 starting at only 35 USD and the necessary accessories (case, fan, monitor, etc ..) it is easily a bargain compared to renting vps for every website every month (10$ per month for a 2GB of Memory machine on DigitalOcean).

Description

According to wikipedia: “The Raspberry Pi is a series of small single-board computers developed in the United Kingdom by the Raspberry Pi Foundation to promote teaching of basic computer science in schools and in developing countries. The original model became far more popular than anticipated, selling outside its target market for uses such as robotics. It does not include peripherals (such as keyboards and mice) or cases. However, some accessories have been included in several official and unofficial bundles.”

Needless to say, people quickly realized that the computer has a lot of potential aside from teaching purposes due to it’s sheer size, lightweight and inexpensive price tag. The Raspberry Pi 4 was announced in June 2019, with 3 memory options: 1Gb, 2Gb, 4Gb. Starting from 35 USD for the 1GB option, the 4GB option goes up to 55 USD, the latter is what I ultimately decide to use for the project.

The launch of the Pi 4 did not go so well, the device suffered CPU throttling, USB C charging failures, booting issues, etc… To solve the problems, I would have to acquire the Flirc Raspberry Pi 4 Case, which is recommended by many users for the reason that it provides active cooling so the computer does not throttle.

Having a quad core cpu (Quad core Cortex-A72 (ARM v8)) makes all the difference when comparing the Pi 4 to other micro-computers. Combined with the 4GBs of RAM, the computer will make good use of its hardware hosting multiple websites and databases.

In addition to the case, the Pi also need sd card due to its lack of internal storage. The Samsung 64GB EVO Plus is one of the best recommended for this system as a result of the perfect balace that it strikes between pricing and performance.

After getting all the hardware needed for the Pi, the last thing it needs is an operating system. CentOs is my choice for many reasons. According to maketecheasier.com:“CentOS is a community built distro based on the source code of RedHat Enterprise Linux (REHL). It is also the most popular distro for web server as of now. The benefit of using CentOS is that it is stable and seldom crashes. If you have some packages that only work with REHL, chances are you can get them to work in CentOS as well. Being the most popular distro, it also has the benefit of great support, either from their community forum, or anywhere on the Web”.

By using Nginx for server software which uses the asynchronous, event‑driven model to handle connections that help takes full use of the multi-core cpu, the Pi can perform significantly better than one using Apache. As a result, the Pi can theoretically process millions of simultaneous processes and scale very well.

Tools and Technologies

Since I am most familiar with the javascript fullstack, it would only be natural for me to setup the system so as to I only have to use javascript without learning any other languages to get the server configured and running.

  • NodeJS: A javascript runtime environment that enable javascript code to be ran on the server.
  • NVM (Node Version Manager): It is recommended that instead of installing Node js directly, one should use NVM to enable version switch on the fly without uninstalling and reinstalling NodeJS.
  • NPM (Node Package Manager): A javascript package manager that is a must have when developing for the javascript stack.
  • Git: One of the best known version control system in the world.
  • Nodemon: A DevDependency tools that can be used with systemctl to help deploy application to production.

Skills Required

The below are knowledge and prior experiences required for this project:

  • Unix commands: Since Linux is unix based and we are not going to install any desktop environment, It is required to navigate the operating system using unix commands.
  • Basic understanding of Networking, Linux, Protocols, SSL Certificate, Security, Authentications, User Role, etc...
  • Troubleshooting skills: One of the most important skill when doing anything. Such as reading log files, locating errors, develop troubleshooting strategies.
  • Javascript, NodeJS, HTTP methods, SSH to connect to the machine remotely, managing services in Linux, Nginx.

Most of the skills required can be learned from the free resources available on the internet for instance:

  • TraversyMedia (javascript related topics)
  • DistroTube (Linux related topics)
  • The various documentation for each technology

Outcome

Should the project be successful, one would gain various new skills and understandings of how the World Wide Web works, the fundamentals of the Linux operating system, the endless potential of micro-computers and much more.

The project should provide us a fast, reliable, easy to deploy and scalable server.
